## 1. Java Introduction & Basics

### What is Java?
Java is a high-level, object-oriented, platform-independent programming language developed by Sun Microsystems (now Oracle) in 1995.

### Features of Java
1. **Simple** - Easy to learn and understand
2. **Object-Oriented** - Everything is an object
3. **Platform Independent** - Write Once, Run Anywhere (WORA)
4. **Secure** - No explicit pointers, runs in sandbox
5. **Robust** - Strong memory management, exception handling
6. **Multithreaded** - Supports concurrent execution
7. **Portable** - Bytecode can run on any platform
8. **High Performance** - JIT compiler optimizes performance

### JDK, JRE, JVM

```
┌─────────────────────────────────────┐
│           JDK (Java Development Kit)│
│  ┌──────────────────────────────┐   │
│  │    JRE (Java Runtime Env)    │   │
│  │  ┌────────────────────────┐  │   │
│  │  │   JVM (Virtual Machine)│  │   │
│  │  └────────────────────────┘  │   │
│  │      + Libraries             │   │
│  └──────────────────────────────┘   │
│      + Development Tools             │
│      (javac, javadoc, jar, etc.)     │
└─────────────────────────────────────┘
```

### First Java Program

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

**Explanation:**
- `public` - access modifier (accessible everywhere)
- `class` - keyword to define a class
- `Hello` - class name (must match filename)
- `main` - entry point of program
- `String[] args` - command line arguments
- `System.out.println()` - prints output

### Compilation and Execution

```bash
# Compile Java file (creates .class file with bytecode)
javac Hello.java

# Run Java program
java Hello
```

### Viva Questions

**Q1: What is JVM?**
A: Java Virtual Machine executes Java bytecode. It provides platform independence and runtime environment.

**Q2: What is bytecode?**
A: Platform-independent intermediate code generated by Java compiler (.class files).

**Q3: Why is Java platform independent?**
A: Java compiler generates bytecode which can run on any platform having JVM.

**Q4: What is the difference between JDK and JRE?**
A: JDK = JRE + Development Tools (compiler, debugger). JRE = JVM + Libraries to run programs.

**Q5: Is Java 100% object-oriented?**
A: No, because Java uses primitive data types (int, char, boolean, etc.) which are not objects.

**Q6: Why is main method static?**
A: So that JVM can call it without creating an object of the class.

**Q7: What is the purpose of public static void main(String[] args)?**
A: It's the entry point. Public - accessible by JVM, static - no object needed, void - returns nothing.

**Q8: Can we have multiple main methods in Java?**
A: Yes, through overloading, but JVM calls only `public static void main(String[] args)`.

### Type Casting

```java
public class TypeCasting {
    public static void main(String[] args) {
        
        // Widening (Implicit/Automatic)
        int i = 100;
        long l = i;      // int to long
        float f = l;     // long to float
        double d = f;    // float to double
        
        System.out.println("Widening: " + d);
        
        // Narrowing (Explicit/Manual)
        double d2 = 100.99;
        int i2 = (int) d2;   // double to int (loses decimal)
        
        System.out.println("Narrowing: " + i2);  // 100
    }
}
```

### Viva Questions

**Q1: How many types of data types in Java?**
A: Two types - Primitive (8 types) and Reference types.

**Q2: What is the size of int in Java?**
A: 4 bytes (32 bits), range: -2,147,483,648 to 2,147,483,647

**Q3: What is the difference between float and double?**
A: float is 4 bytes (6-7 decimal digits), double is 8 bytes (15 decimal digits). Double has higher precision.

**Q4: Why do we use 'L' with long and 'f' with float?**
A: To specify the literal type. Without L, number is treated as int. Without f, it's treated as double.

**Q5: What is type casting?**
A: Converting one data type to another. Widening (automatic), Narrowing (explicit with data loss).

**Q6: What is the default value of instance variables?**
A: int/byte/short/long: 0, float/double: 0.0, boolean: false, char: '\u0000', reference: null

**Q7: What is the difference between instance and static variables?**
A: Instance variables belong to object (different for each object). Static variables belong to class (shared by all objects).

**Q8: Can we change the value of a final variable?**
A: No, final variables are constants and cannot be modified after initialization.

## 3. Operators

### Types of Operators

```java
public class Operators {
    public static void main(String[] args) {
        
        int a = 10, b = 20;
        
        // 1. Arithmetic Operators
        System.out.println("a + b = " + (a + b));   // 30
        System.out.println("a - b = " + (a - b));   // -10
        System.out.println("a * b = " + (a * b));   // 200
        System.out.println("b / a = " + (b / a));   // 2
        System.out.println("b % a = " + (b % a));   // 0 (remainder)
        
        // 2. Relational Operators
        System.out.println("a == b: " + (a == b));  // false
        System.out.println("a != b: " + (a != b));  // true
        System.out.println("a > b: " + (a > b));    // false
        System.out.println("a < b: " + (a < b));    // true
        System.out.println("a >= b: " + (a >= b));  // false
        System.out.println("a <= b: " + (a <= b));  // true
        
        // 3. Logical Operators
        boolean x = true, y = false;
        System.out.println("x && y: " + (x && y));  // false (AND)
        System.out.println("x || y: " + (x || y));  // true (OR)
        System.out.println("!x: " + (!x));          // false (NOT)
        
        // 4. Assignment Operators
        int c = 10;
        c += 5;  // c = c + 5
        System.out.println("c += 5: " + c);  // 15
        c -= 3;  // c = c - 3
        System.out.println("c -= 3: " + c);  // 12
        c *= 2;  // c = c * 2
        System.out.println("c *= 2: " + c);  // 24
        c /= 4;  // c = c / 4
        System.out.println("c /= 4: " + c);  // 6
        c %= 4;  // c = c % 4
        System.out.println("c %= 4: " + c);  // 2
        
        // 5. Unary Operators
        int d = 10;
        System.out.println("d++: " + (d++));  // 10 (post-increment)
        System.out.println("d: " + d);        // 11
        System.out.println("++d: " + (++d));  // 12 (pre-increment)
        System.out.println("d--: " + (d--));  // 12 (post-decrement)
        System.out.println("--d: " + (--d));  // 10 (pre-decrement)
        
        // 6. Ternary Operator
        int max = (a > b) ? a : b;
        System.out.println("Max: " + max);    // 20
        
        // 7. Bitwise Operators
        int p = 5, q = 3;  // 5 = 0101, 3 = 0011
        System.out.println("p & q: " + (p & q));   // 1 (0001)
        System.out.println("p | q: " + (p | q));   // 7 (0111)
        System.out.println("p ^ q: " + (p ^ q));   // 6 (0110)
        System.out.println("~p: " + (~p));         // -6 (1010)
        System.out.println("p << 1: " + (p << 1)); // 10 (left shift)
        System.out.println("p >> 1: " + (p >> 1)); // 2 (right shift)
    }
}
```

### Viva Questions

**Q1: What is the difference between ++ and -- operators?**
A: ++ increments by 1, -- decrements by 1. Pre (++i) increments first then uses value. Post (i++) uses value then increments.

**Q2: What is the difference between == and equals()?**
A: == compares memory addresses (for objects) or values (primitives). equals() compares content/values of objects.

**Q3: What is the difference between & and &&?**
A: & is bitwise AND (evaluates both sides). && is logical AND (short-circuit, stops if first is false).

**Q4: What is ternary operator?**
A: Shorthand for if-else: `condition ? value_if_true : value_if_false`

**Q5: What is the precedence of operators?**
A: () > unary > arithmetic > relational > logical > assignment

### Loops

```java
public class Loops {
    public static void main(String[] args) {
        
        // 1. For Loop
        System.out.println("For Loop:");
        for (int i = 1; i <= 5; i++) {
            System.out.print(i + " ");
        }
        System.out.println();
        
        // 2. While Loop
        System.out.println("While Loop:");
        int j = 1;
        while (j <= 5) {
            System.out.print(j + " ");
            j++;
        }
        System.out.println();
        
        // 3. Do-While Loop
        System.out.println("Do-While Loop:");
        int k = 1;
        do {
            System.out.print(k + " ");
            k++;
        } while (k <= 5);
        System.out.println();
        
        // 4. Enhanced For Loop (for-each)
        System.out.println("For-each Loop:");
        int[] arr = {10, 20, 30, 40, 50};
        for (int num : arr) {
            System.out.print(num + " ");
        }
        System.out.println();
        
        // 5. Break statement
        System.out.println("Break:");
        for (int i = 1; i <= 10; i++) {
            if (i == 6) break;
            System.out.print(i + " ");
        }
        System.out.println();
        
        // 6. Continue statement
        System.out.println("Continue:");
        for (int i = 1; i <= 10; i++) {
            if (i % 2 == 0) continue;
            System.out.print(i + " ");  // prints odd numbers
        }
        System.out.println();
    }
}
```

### Viva Questions

**Q1: What is the difference between while and do-while?**
A: while checks condition first (may not execute). do-while executes at least once then checks condition.

**Q2: What is the difference between break and continue?**
A: break exits the loop completely. continue skips current iteration and moves to next.

**Q3: Can we use switch without break?**
A: Yes, but it will cause fall-through (executes all cases after match until break is found).

**Q4: What types can be used in switch?**
A: byte, short, int, char, String (Java 7+), enum

**Q5: What is enhanced for loop?**
A: For-each loop to iterate over arrays/collections without index: `for(type var : array)`

```java
import java.util.Arrays;

public class ArrayOperations {

        System.out.println("Arrays equal: " + equal);
        
        // 6. Sum of elements
        int sum = 0;
        for (int num : arr) {
            sum += num;
        }
        System.out.println("Sum: " + sum);
        
        // 7. Finding max and min
        int max = arr[0], min = arr[0];
        for (int num : arr) {
            if (num > max) max = num;
            if (num < min) min = num;
        }
```

### Multi-dimensional Arrays

```java
public class MultiDimensionalArray {
    public static void main(String[] args) {
        
        // 2D Array
        int[][] matrix = {
            {1, 2, 3},
            {4, 5, 6},
            {7, 8, 9}
```

## 6. Strings

### String Basics

```java
public class StringBasics {
    public static void main(String[] args) {
        
        // 1. Creating Strings
        String s1 = "Hello";           // String literal (goes to string pool)
        String s2 = new String("Hello"); // Using new (creates new object)
        String s3 = "Hello";           // Reuses from string pool
        
        // 2. String comparison
        System.out.println("s1 == s2: " + (s1 == s2));       // false (different objects)
        System.out.println("s1 == s3: " + (s1 == s3));       // true (same reference)
        System.out.println("s1.equals(s2): " + s1.equals(s2)); // true (same content)
        
        // 3. String is immutable
        String str = "Java";
        str.concat(" Programming");  // creates new string but doesn't change str
        System.out.println("str: " + str);  // still "Java"
        
        str = str.concat(" Programming");  // assign to modify reference
        System.out.println("str: " + str);  // now "Java Programming"
    }
}
```

### String Pool

```java
public class StringPool {
    public static void main(String[] args) {
        
        // String pool (constant pool)
        String s1 = "Java";    // goes to string pool
        String s2 = "Java";    // reuses from pool
        String s3 = new String("Java");  // creates in heap
        
        System.out.println("s1 == s2: " + (s1 == s2));  // true (same reference)
        System.out.println("s1 == s3: " + (s1 == s3));  // false (different objects)
        
        // intern() method - adds to pool if not present
        String s4 = s3.intern();
        System.out.println("s1 == s4: " + (s1 == s4));  // true (now from pool)
    }
}
```

### Viva Questions

**Q1: What is String in Java?**
A: String is a sequence of characters. It's a class in Java, and objects are immutable.

**Q2: Why are Strings immutable?**
A: For security, synchronization, performance (string pool, caching, hashcode).

**Q3: What is String pool?**
A: Special memory area in heap where String literals are stored to optimize memory.

**Q4: What is the difference between == and equals() for Strings?**
A: == compares references (memory addresses), equals() compares content.

**Q5: What is the difference between String s1 = "Hello" and String s2 = new String("Hello")?**
A: First creates in string pool, second creates new object in heap memory.

**Q6: How to compare strings?**
A: Using equals() for content, compareTo() for lexicographic order, == for references.

**Q7: What is the use of intern() method?**
A: Adds string to string pool if not present and returns reference from pool.

**Q8: Can we modify a String object?**
A: No, Strings are immutable. Any modification creates a new String object.

```java
        sb.replace(0, 5, "Hi");
        System.out.println("After replace: " + sb);  // Hi, World
        
        // 5. Delete (removes characters in range)
        sb.delete(2, 4);
        System.out.println("After delete: " + sb);  // HiWorld
        
        // 6. Reverse
        sb.reverse();
        System.out.println("After reverse: " + sb);  // dlroWiH
        sb.reverse();  // reverse back
        
        // 7. Capacity and Length
        StringBuilder sb2 = new StringBuilder();
        System.out.println("Initial capacity: " + sb2.capacity());  // 16 (default)
        System.out.println("Length: " + sb2.length());  // 0
        
        // 8. Delete character at index
        sb.deleteCharAt(0);
        System.out.println("After deleteCharAt: " + sb);
        
        // 9. Convert to String
        String str = sb.toString();
        System.out.println("Converted to String: " + str);
        
        // 10. Set character at index
        sb.setCharAt(0, 'J');
        System.out.println("After setCharAt: " + sb);
    }
}
```

### Performance Comparison

```java
public class StringPerformance {
    public static void main(String[] args) {
        
        // String (slow for concatenation)
        long start = System.currentTimeMillis();
        String s = "";
        for (int i = 0; i < 10000; i++) {
            s += "a";  // creates new object each time
        }
        long end = System.currentTimeMillis();
        System.out.println("String time: " + (end - start) + "ms");
        
        // StringBuilder (fast, not synchronized)
        start = System.currentTimeMillis();
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 10000; i++) {
            sb.append("a");  // modifies same object
        }
        end = System.currentTimeMillis();
        System.out.println("StringBuilder time: " + (end - start) + "ms");
        
        // StringBuffer (slower than StringBuilder due to synchronization)
        start = System.currentTimeMillis();
        StringBuffer sbf = new StringBuffer();
        for (int i = 0; i < 10000; i++) {
            sbf.append("a");
        }
        end = System.currentTimeMillis();
        System.out.println("StringBuffer time: " + (end - start) + "ms");
    }
}
```

### String vs StringBuilder vs StringBuffer

| Feature | String | StringBuilder | StringBuffer |
|---------|---------|---------------|--------------|
| Mutability | Immutable | Mutable | Mutable |
| Thread-Safe | Yes | No | Yes (Synchronized) |
| Performance | Slow for concatenation | Fast | Slower than StringBuilder |
| Memory | Creates new object | Modifies same object | Modifies same object |
| When to use | When no modification | Single-threaded | Multi-threaded |

### Viva Questions

**Q1: What is the difference between String and StringBuilder?**
A: String is immutable (cannot be changed), StringBuilder is mutable (can be modified).

**Q2: When should we use StringBuilder?**
A: When performing many string modifications (concatenation, append) in single-threaded environment.

**Q3: What is the difference between StringBuilder and StringBuffer?**
A: StringBuilder is not thread-safe (faster), StringBuffer is thread-safe (synchronized, slower).

**Q4: What is the default capacity of StringBuilder?**
A: 16 characters. Formula for new capacity: (old capacity * 2) + 2

**Q5: Why is String immutable but StringBuilder is mutable?**
A: String is designed for security and optimization. StringBuilder is designed for performance with modifications.

**Q6: Can we convert StringBuilder to String?**
A: Yes, using toString() method.

**Q7: Which is faster for string concatenation?**
A: StringBuilder > StringBuffer > String (for multiple concatenations)

**Q8: What happens when StringBuilder capacity is exceeded?**
A: It automatically increases capacity using formula: (old capacity * 2) + 2

## 10. Method Overloading

### Method Overloading Examples

```java
class Calculator {
    
    // Method overloading - same name, different parameters
    
    // 1. Different number of parameters
    int add(int a, int b) {
        return a + b;
    }
    
    int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // 2. Different type of parameters
    double add(double a, double b) {
        return a + b;
    }
    
    String add(String a, String b) {
        return a + b;
    }
    
    // 3. Different order of parameters
    void display(int a, String b) {
        System.out.println("int, String: " + a + ", " + b);
    }
    
    void display(String a, int b) {
        System.out.println("String, int: " + a + ", " + b);
    }
}

public class MethodOverloading {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        System.out.println("add(10, 20): " + calc.add(10, 20));
        System.out.println("add(10, 20, 30): " + calc.add(10, 20, 30));
        System.out.println("add(10.5, 20.5): " + calc.add(10.5, 20.5));
        System.out.println("add(\"Hello\", \"World\"): " + calc.add("Hello", "World"));
        
        calc.display(10, "Java");
        calc.display("Java", 10);
    }
}
```

### Type Promotion in Overloading

```java
class TypePromotion {
    
    void show(int a) {
        System.out.println("int: " + a);
    }
    
    void show(long a) {
        System.out.println("long: " + a);
    }
    
    void show(double a) {
        System.out.println("double: " + a);
    }
}

public class TypePromotionEx {
    public static void main(String[] args) {
        TypePromotion obj = new TypePromotion();
        
        obj.show(10);      // exact match - calls int
        obj.show(100L);    // exact match - calls long
        obj.show(10.5);    // exact match - calls double
        
        byte b = 10;
        obj.show(b);       // byte promoted to int
        
        float f = 10.5f;
        obj.show(f);       // float promoted to double
    }
}
```

### Viva Questions

**Q1: What is method overloading?**
A: Multiple methods with same name but different parameters in the same class.

**Q2: What are the ways to overload a method?**
A: By changing: 1) Number of parameters, 2) Type of parameters, 3) Order of parameters

**Q3: Can we overload methods by changing only the return type?**
A: No, return type alone cannot differentiate overloaded methods.

**Q4: What is compile-time polymorphism?**
A: Method overloading is compile-time polymorphism. Method to call is decided at compile time.

**Q5: Can we overload main method?**
A: Yes, but JVM calls only `public static void main(String[] args)`.

**Q6: Can we overload static methods?**
A: Yes, static methods can be overloaded.

**Q7: What is type promotion in overloading?**
A: When exact match is not found, smaller types are promoted to larger types (byte→int→long→float→double).

**Q8: Can we overload private methods?**
A: Yes, private methods can be overloaded within the same class.

**Q9: What is the advantage of method overloading?**
A: Increases readability, provides same method name for similar operations.

**Q10: What is ambiguity in method overloading?**
A: When compiler cannot decide which method to call due to type promotion or implicit conversion.

```java
public class Palindrome {
    public static boolean isPalindrome(String str) {
        int left = 0, right = str.length() - 1;
        
        while (left < right) {
            if (str.charAt(left) != str.charAt(right)) {
                return false;
            }
            left++;
            right--;
        }
        return true;
    }
```
